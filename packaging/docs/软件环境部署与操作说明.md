# 智慧图书馆系统软件环境部署与操作说明

## 1. 系统说明

本系统为智慧图书馆管理平台，采用前后端分离架构：

- 后端：`backend-library/`，基于 Spring Boot 3.4、Java 17、MySQL 8、Redis
- Web 前端：`front_library/`，基于 Next.js 15、React 18、TypeScript
- Android 客户端：`front-android/`，基于 Expo SDK 53、React Native 0.79、TypeScript
- 微信小程序：`miniapp/`，基于微信小程序原生工程

系统覆盖读者端与管理端，主要功能包括馆藏查询、借阅、预约、评论、罚款、通知，以及后台图书管理、借阅管理、预约管理、用户管理和 RBAC 权限管理。

## 3. 软件环境要求

### 3.1 推荐环境

| 项目 | 推荐版本 | 说明 |
| --- | --- | --- |
| 操作系统 | Windows 10/11 或 Linux | 仓库脚本优先适配 Windows PowerShell |
| JDK | 17 | 后端运行必需 |
| Node.js | 20 LTS | 前端推荐版本 |
| npm | 10+ | 前端依赖管理 |
| MySQL | 8.x | 核心业务数据库 |
| Redis | 7.x | 可选，本地开发可关闭 |
| Android Studio / Android SDK | 较新稳定版 | Android 客户端调试与打包 |
| 微信开发者工具 | 较新稳定版 | 小程序运行与预览 |
| Docker Desktop / Docker Engine | 较新稳定版 | 推荐用于一键部署 |

### 3.2 端口规划

| 服务 | 默认端口 | 说明 |
| --- | --- | --- |
| 前端 | 3000 | Next.js Web 服务 |
| 后端 | 8089 | Spring Boot API 服务 |
| MySQL | 3306 | Docker 默认端口 |
| Redis | 6379 | Docker 默认端口 |

说明：

- 后端本地默认配置文件中使用的数据库地址是 `localhost:3307`。
- 如果你的本地 MySQL 实际运行在 `3306`，启动后端前需要通过环境变量覆盖 `SPRING_DATASOURCE_URL`。

## 4. 数据库脚本说明

`database/` 目录提供三类 SQL 文件：

- `create.sql`
  用途：基础建库建表脚本，适合全新环境快速初始化。
- `library_management_linux_init.sql`
  用途：完整初始化脚本，包含表结构、角色权限、示例用户和演示数据，适合论文答辩、功能演示和 Linux 服务器导入。
- `migrate-legacy-schema.sql`
  用途：旧版数据库升级到当前结构时使用。

使用建议：

- 本地开发或 Docker 首次部署：优先使用 `create.sql`
- 演示环境或需要开箱即用数据：优先使用 `library_management_linux_init.sql`
- 不要在同一个空数据库中同时导入 `create.sql` 和 `library_management_linux_init.sql`

## 5. Docker 部署方式

Docker 部署适合快速演示和统一环境交付。

### 5.1 准备工作

1. 安装 Docker Desktop 或 Docker Engine。
2. 在交付包根目录将 `.env.example` 复制为 `.env`。
3. 根据实际需要修改以下配置：

- `MYSQL_ROOT_PASSWORD`
- `MYSQL_DATABASE`
- `MYSQL_USER`
- `MYSQL_PASSWORD`
- `JWT_SECRET`
- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`

### 5.2 启动命令

在交付包根目录执行：

```powershell
docker compose up -d --build
```

### 5.3 部署说明

- MySQL 容器首次启动时会自动执行 `backend-library/create.sql`
- Redis 容器默认开启
- 后端容器默认连接 Docker 网络中的 MySQL 与 Redis
- 前端容器通过 `BACKEND_URL=http://backend:8089` 访问后端

### 5.4 访问地址

- 前端首页：`http://localhost:3000`
- 后端接口：`http://localhost:8089/api`
- 后端健康检查：`http://localhost:8089/actuator/health`

### 5.5 常用 Docker 运维命令

启动：

```powershell
docker compose up -d --build
```

查看日志：

```powershell
docker compose logs -f backend
docker compose logs -f frontend
```

停止：

```powershell
docker compose down
```

删除容器和匿名网络：

```powershell
docker compose down --remove-orphans
```

重置数据库卷：

```powershell
docker compose down -v
```

说明：执行 `docker compose down -v` 会清空数据库和 Redis 数据，请谨慎操作。

## 6. 本地手动部署方式

本地部署适合代码调试、二次开发和教学演示。

### 6.1 数据库初始化

先启动 MySQL 8 服务，然后任选一种方式初始化数据库。

#### 方案 A：基础建表

```powershell
cd backend-library
mysql -u root -p < create.sql
```

#### 方案 B：导入完整演示数据

```powershell
cd backend-library
mysql --default-character-set=utf8mb4 -u root -p < library_management_linux_init.sql
```

### 6.2 启动后端

#### 情况一：本地 MySQL 使用 3307 端口

直接执行：

```powershell
cd backend-library
.\mvnw.cmd spring-boot:run
```

#### 情况二：本地 MySQL 使用 3306 端口

先设置环境变量，再启动：

```powershell
cd backend-library
$env:SPRING_DATASOURCE_URL="jdbc:mysql://localhost:3306/library_management?createDatabaseIfNotExist=true&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&characterEncoding=utf8"
$env:SPRING_DATASOURCE_USERNAME="root"
$env:SPRING_DATASOURCE_PASSWORD="root"
$env:APP_SECURITY_REDIS_ENABLED="false"
$env:APP_CACHE_REDIS_ENABLED="false"
.\mvnw.cmd spring-boot:run
```

说明：

- 如果本地没有 Redis，可将 `APP_SECURITY_REDIS_ENABLED` 和 `APP_CACHE_REDIS_ENABLED` 设置为 `false`
- 后端默认监听 `8089` 端口

### 6.3 启动前端

```powershell
cd front_library
npm install
npm run dev
```

默认访问地址：

- 前端：`http://localhost:3000`
- 后端代理转发目标：`http://localhost:8089/api`

### 6.4 启动 Android 客户端

#### 环境准备

- 安装 Node.js 20 LTS
- 安装 JDK 17
- 安装 Android Studio 或 Android SDK
- 准备 Android 模拟器或真机

#### 配置接口地址

先复制环境变量模板：

```powershell
cd front-android
Copy-Item .env.example .env
```

然后根据实际联调环境修改 `EXPO_PUBLIC_API_BASE_URL`：

- Android Studio 模拟器：`http://10.0.2.2:8089/api`
- 真机调试：改为宿主机局域网 IP，例如 `http://192.168.1.10:8089/api`

注意：

- 不要把真机接口地址写成 `localhost`
- 如果后端不是 `8089` 端口，请同步修改

#### 安装依赖并启动

```powershell
cd front-android
npm install
npm run start
```

启动 Android 调试：

```powershell
cd front-android
npm run android
```

#### 生成 APK

调试包：

```powershell
cd front-android
npm run android:debug
```

输出路径：

```text
front-android/android/app/build/outputs/apk/debug/app-debug.apk
```

发布包：

```powershell
cd front-android
npm run android:release
```

说明：

- 发布包签名前需按 `front-android/README.md` 配置 keystore 参数
- 若只做本地演示，可优先使用 debug 包

### 6.5 启动微信小程序

#### 方式一：直接打开小程序目录

在微信开发者工具中打开：

```text
SmartLibrarySystem_Delivery_YYYYMMDD\miniapp
```

#### 方式二：打开交付包根目录

交付包根目录包含 `project.config.json`，已将 `miniprogramRoot` 指向 `miniapp/`，因此也可直接打开交付包根目录。

#### 配置后端地址

小程序接口地址位于：

```text
miniapp/config/env.js
```

默认值示例：

```javascript
const DEFAULT_API_BASE_URL = 'http://154.19.43.33:8089/api'
```

联调建议：

- 本地模拟器调试：改成局域网可访问地址
- 真机预览：必须使用手机可访问的局域网或公网地址
- 不要使用 `127.0.0.1` 或 `localhost`

#### 运行步骤

1. 先启动后端服务
2. 在微信开发者工具中导入 `miniapp/`
3. 修改 `config/env.js` 中的 API 地址
4. 点击“编译”或“预览”

### 6.6 Windows 快速启动脚本

交付包根目录提供以下脚本：

- `start.ps1`：同时启动前后端
- `start-all.bat`：调用 `start.ps1`
- `start-backend-only.bat`：只启动后端
- `seed-backend-data.bat`：调用后端种子数据脚本

如果系统已正确安装 Java、Node.js 和 MySQL，可以直接在交付包根目录执行：

```powershell
.\start.ps1
```

说明：

- 该脚本只负责启动 Web 前端和后端
- Android 和小程序需按各自章节手动启动

## 8. 演示数据生成

如果使用 `create.sql` 只初始化了空库，可在后端启动后补充测试数据：

```powershell
cd backend-library
.\seed-backend-data.ps1
```

也可指定生成规模：

```powershell
cd backend-library
.\seed-backend-data.ps1 -Users 30 -Books 80
```

脚本默认会尝试使用管理员账号 `admin / admin123` 登录，再调用开发环境接口生成数据。

## 10. 常见问题

### 10.1 前端可以打开，但接口全部失败

请检查：

- 后端是否已启动在 `8089` 端口
- `front_library/next.config.js` 中代理目标是否被环境变量 `BACKEND_URL` 覆盖
- 浏览器是否存在旧 token，可尝试退出后重新登录

### 10.2 Android 客户端无法连到后端

请检查：

- `.env` 中 `EXPO_PUBLIC_API_BASE_URL` 是否正确
- Android 模拟器是否使用了 `10.0.2.2`
- 真机是否使用可访问宿主机的局域网 IP
- 后端 CORS 和网络是否允许访问

### 10.3 小程序无法请求本地接口

请检查：

- `miniapp/config/env.js` 是否改成了正确地址
- 该地址是否能被微信开发者工具或手机访问
- 服务端是否已经启动

### 10.4 后端启动时报数据库连接失败

最常见原因是数据库端口不一致：

- 本地配置默认使用 `3307`
- Docker 默认暴露 `3306`

若实际数据库在 `3306`，请按本说明第 6.2 节覆盖 `SPRING_DATASOURCE_URL`。

### 10.5 没有 Redis 是否能运行

可以。本地开发场景下，将以下环境变量设置为 `false` 即可：

```powershell
$env:APP_SECURITY_REDIS_ENABLED="false"
$env:APP_CACHE_REDIS_ENABLED="false"
```

### 10.6 如何恢复到全新数据状态

- Docker 部署：执行 `docker compose down -v` 后重新启动
- 本地部署：删除数据库后重新导入 `create.sql` 或 `library_management_linux_init.sql`
