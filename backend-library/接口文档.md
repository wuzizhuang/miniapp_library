# 图书管理系统 - 接口文档 (API Definitions)

本文档基于系统最新开发需求编写，提供给前端与移动端调用参考。采用标准的RESTful风格。

## 1. 基础规范说明
- **Base URL**: `/api`
- **认证方式**: JWT Bearer Token. 需在HTTP Header携带：`Authorization: Bearer <your_token>`
- **数据交互格式**: `application/json`
- **分页规范**: 列表查询接口默认支持 `page` (页码, 从0开始) 和 `size` (每页条数, 默认10). 返回包含 `content`, `totalElements`, `totalPages` 等字段。

## 2. 认证与个人中心模块 (Auth & Profile)

### 2.1 用户注册 / 登录
- **`POST /api/auth/register`**
  - **描述**: 新用户注册。
  - **Body**: `{ "username": "", "password": "", "email": "", "fullName": "" }`
- **`POST /api/auth/login`**
  - **描述**: 用户登录换取Token。
  - **Body**: `{ "username": "", "password": "" }`
  - **Response**: `{ "token": "jwt_string", "userId": 1, "username": "", "role": "USER|ADMIN|...", "roles": ["USER"], "permissions": [] }`
- **`POST /api/auth/logout`**
  - **描述**: 登出当前会话。
  - **边界**: 当前实现仅清理服务端 `SecurityContext`，JWT 不会立即失效，会继续有效直到过期。
- **`GET /api/auth/me`**
  - **描述**: 获取当前登录用户基础信息。
- **`GET /api/auth/context`**
  - **描述**: 获取当前登录用户的角色与权限上下文，用于前端 RBAC 渲染。

### 2.2 个人资料
- **`GET /api/users/me/profile`**
  - **描述**: 获取当前登录用户完整个人资料。
- **`PUT /api/users/me/profile`**
  - **描述**: 更新当前登录用户个人资料。

### 2.3 找回密码
- **`POST /api/auth/forgot-password`**
  - **描述**: 发起找回密码请求。
  - **Body**: `{ "email": "user@example.com" }`
  - **Response**: `{ "message": "...", "deliveryMethod": "EMAIL|LOG", "expiresInMinutes": 30 }`
  - **安全说明**: 无论邮箱是否存在，响应保持一致，不泄露账号存在性。
- **`GET /api/auth/reset-password/validate?token=...`**
  - **描述**: 校验重置 token 是否仍然有效。
  - **Response**: `{ "valid": true|false, "message": "..." }`
- **`POST /api/auth/reset-password`**
  - **描述**: 使用合法 token 提交新密码。
  - **Body**: `{ "token": "...", "password": "NewPassword123!" }`
  - **Response**: `{ "message": "...", "deliveryMethod": "NONE", "expiresInMinutes": null }`
  - **当前边界**:
  - 当前版本未接邮件基础设施，重置链接通过后端日志输出，供本地联调使用。
  - token 为一次性、带有效期，服务端仅保存哈希值。

## 3. 图书编目模块 (Catalog)
管理书目（SPU）、馆藏副本（SKU）、分类、出版社及作者信息。

### 3.1 图书 (Books)
- **`GET /api/books`**
  - **描述**: 分页检索图书（支持按分类ID、标题、ISBN、作者、出版社模糊查询）。
- **`GET /api/books/{id}`**
  - **描述**: 获取单本图书的详细信息（包含关联的副本列表及作者列表）。
- **`POST /api/books`** | **`PUT /api/books/{id}`** | **`DELETE /api/books/{id}`**
  - **描述**: 增删改书目信息（限Admin）。

### 3.2 馆藏副本 (Book Copies)
- **`GET /api/book-copies`**
  - **描述**: 条件检索所有馆藏副本。
- **`POST /api/book-copies`**
  - **描述**: 入库新书副本（限Admin）。
  - **Body**: `{ "bookId": 1, "price": 45.00, "acquisitionDate": "...", "locationCode": "A-1" }`

### 3.3 分类 (Categories)
- **`GET /api/categories`**
  - **描述**: 分页获取分类列表。
- **`POST /api/categories`** | **`PUT /api/categories/{id}`** | **`DELETE /api/categories/{id}`**
  - **描述**: 分类节点维护（限Admin）。

### 3.4 出版社 (Publishers) / 作者 (Authors)
- **`GET /api/publishers`** | **`GET /api/authors`** (及相应的增删改查)
  - **描述**: 获取或管理实体元数据池。

## 4. 流通与借阅模块 (Circulation)

### 4.1 借还与续借
- **`GET /api/loans`** / **`GET /api/loans/{id}`**
  - **描述**: 查询借阅单（管理员查全部，特定用户可查自己的借阅记录）。
- **`POST /api/loans`**
  - **描述**: 办理借书。
  - **Body**: `{ "copyId": 1 }`
- **`PUT /api/loans/{id}/return`**
  - **描述**: 办理还书。
- **`PUT /api/loans/{id}/renew`**
  - **描述**: 办理续借（系统校验是否达最多次数或有人排队预约）。
- **`PUT /api/loans/{id}/lost`**
  - **描述**: 标记图书遗失，自动生成罚款单。

### 4.2 逾期查询
- **`GET /api/loans/overdue`**
  - **描述**: 获取所有当前处于逾期未还状态的借阅记录。

---

## 5. 预约与排队模块 (Reservation)

- **`POST /api/reservations`**
  - **描述**: 发起预约请求；当有可分配副本时会直接进入待取书状态，否则进入排队状态。
  - **Body**: `{ "bookId": 102 }`
- **`GET /api/reservations`**
  - **描述**: 管理员分页查询全部预约，可按状态筛选。
- **`GET /api/reservations/me`**
  - **描述**: 查询当前登录用户的所有预约单及排队进度。
- **`PUT /api/reservations/{id}/cancel`**
  - **描述**: 取消预约。
- **`PUT /api/reservations/{id}/fulfill`**
  - **描述**: 预约履约处理（书籍到馆且该用户来取时调用，限Admin）。

---

## 6. 信用与罚款模块 (Fines)

- **`GET /api/fines`** / **`GET /api/fines/me`**
  - **描述**: 获取罚款单列表；`/me` 为当前用户，`/api/fines` 为管理员分页查询。
  - **补充**: 管理员列表支持按状态筛选，并按罚款 ID、用户名、邮箱、读者姓名、书名、罚款原因进行关键词搜索。
- **`GET /api/fines/{fineId}`**
  - **描述**: 获取单笔罚款详情（管理员或罚款归属用户）。
- **`GET /api/fines/user/{userId}`**
  - **描述**: 获取指定用户罚款列表（管理员）。
- **`POST /api/fines/{fineId}/pay`**
  - **描述**: 缴纳罚款。
- **`POST /api/fines/{fineId}/waive`**
  - **描述**: 豁免罚款（管理员）。

---

## 7. 消息中心与通知模块 (Notifications)

- **`GET /api/notifications`**
  - **描述**: 分页获取当前用户的站内信或系统通知。
- **`GET /api/notifications/unread-count`**
  - **描述**: 获取未读消息总数。
- **`PUT /api/notifications/{id}/read`**
  - **描述**: 标记单条消息为已读。
- **`PUT /api/notifications/read-all`**
  - **描述**: 一键全部已读。
- **`DELETE /api/notifications/{id}`**
  - **描述**: 删除单条通知。
- **`DELETE /api/notifications/read`**
  - **描述**: 删除当前用户所有已读通知。
- **边界**:
  - 当前通知 DTO 会提供 `targetType`、`routeHint`、`businessKey`。
  - `targetId` 当前未稳定落具体业务实体，前端只能完成“跳到相关页面”的最小闭环。

## 8. 评论与审核 (Reviews)

- **`POST /api/reviews`**
  - **描述**: 创建评论。
- **`GET /api/reviews/me`**
  - **描述**: 获取当前用户评论列表。
- **`PUT /api/reviews/{id}`**
  - **描述**: 更新评论。
- **`DELETE /api/reviews/{id}`**
  - **描述**: 删除评论。
- **`GET /api/admin/reviews/pending`**
  - **描述**: 获取待审核评论分页列表（管理员）。
- **`PUT /api/admin/reviews/{id}/audit?approved=true|false`**
  - **描述**: 审核通过或驳回评论（管理员）。

---

## 9. 系统管理与统计面板 (Admin & System)

### 9.1 统计报表
- **`GET /api/admin/dashboard/stats`**
  - **描述**: 获取全局运行状态统计（藏书量、流通量、正在借阅数、逾期数等），用于绘制看板大屏。
  
### 9.2 测试辅助接口与配置接口
- **`GET /api/public/health`**
  - **描述**: 健康检查探针（开放端点）。
- **`POST /api/admin/fix/direct-create-admin`** / **`POST /api/admin/fix/fix-enum-case`**
  - **描述**: 系统工具接口（数据修正修复）。
- **`POST /api/dev/seed`** / **`POST /api/dev/reset-test-data`**
  - **描述**: 仅开发环境开启，重置系统和种子灌录。
