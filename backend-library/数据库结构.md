# 图书管理系统 - 数据库结构文档

本文档基于系统最新的开发需求和底层实现编写，描述了整个系统的核心数据模型及其关联关系。

## 1. 核心实体表

### 1.1 用户表 (`users`)
存储系统的登录用户（包括读者与管理员），已扩展画像维度以支持个性化推荐。
| 字段名 | 类型 | 约束 | 描述 |
| ------ | --- | --- | --- |
| `user_id` | INT | PK, Auto Increment | 用户唯一标识 |
| `username` | VARCHAR(50) | UNIQUE, NOT NULL | 登录用户名 |
| `password_hash` | VARCHAR(255)| NOT NULL | 密码哈希串 |
| `email` | VARCHAR(100)| UNIQUE, NOT NULL | 邮箱地址 |
| `full_name` | VARCHAR(100)| NOT NULL | 真实姓名 |
| `role` | ENUM | 'admin', 'user' (默认 user) | 角色权限 |
| `status` | ENUM | 'active', 'inactive' (默认 active)| 账户状态 |
| `department` | VARCHAR(100)| NULL | 院系/部门 (画像维度) |
| `major` | VARCHAR(100)| NULL | 专业 (画像维度) |
| `identity_type`| ENUM | student/teacher/staff/visitor(默认 student) | 身份类型 (画像维度) |
| `enrollment_year`| YEAR | NULL | 入学/入职年份 (画像维度) |
| `interest_tags` | JSON | NULL | 兴趣标签 (如：["科幻", "Python"]) |
| `create_time` | TIMESTAMP | NOT NULL | 创建时间 |
| `update_time` | TIMESTAMP | NOT NULL | 更新时间 |

### 1.2 图书表 (`books`)
图书的基本信息（SPU级），描述书籍本身的元数据。
| 字段名 | 类型 | 约束 | 描述 |
| ------ | --- | --- | --- |
| `book_id` | INT | PK, Auto Increment | 图书唯一标识 |
| `isbn` | VARCHAR(20) | UNIQUE, NOT NULL | 国际标准书号 |
| `title` | VARCHAR(255)| NOT NULL | 书名 |
| `cover_url` | VARCHAR(255)| NULL | 封面图片链接 |
| `description` | TEXT | NULL | 内容简介 |
| `page_count` | INT | NULL | 页数 |
| `published_year`| INT | NOT NULL | 出版年份 |
| `language` | VARCHAR(50) | NOT NULL | 语言种类 |
| `publisher_id` | INT | FK (publishers.publisher_id) | 出版社外键 |
| `category_id` | INT | FK (categories.category_id) | 分类外键 |

### 1.3 馆藏副本表 (`book_copies`)
对应实体的纸质书（SKU级），融合了物联网和空间感知特性。
| 字段名 | 类型 | 约束 | 描述 |
| ------ | --- | --- | --- |
| `copy_id` | INT | PK, Auto Increment | 副本唯一标识（条码号） |
| `book_id` | INT | FK (books.book_id) | 关联图书标识 |
| `status` | ENUM | available/borrowed/reserved/lost/damaged | 当前在库/借阅状态 |
| `acquisition_date`| DATE | NOT NULL | 入库日期 |
| `price` | DECIMAL(10,2)| NOT NULL | 购入价格 |
| `notes` | TEXT | NULL | 破损等备注说明 |
| `location_code` | VARCHAR(50) | NULL | 索书号/物理位置 |
| `rfid_tag` | VARCHAR(64) | UNIQUE | RFID唯一标识串（自助借还） |
| `floor_plan_id` | INT | NULL | 关联楼层地图ID |

## 2. 核心业务表

### 2.1 借阅记录表 (`loans`)
包含借还过程记录与隐性反馈。
| 字段名 | 类型 | 约束 | 描述 |
| ------ | --- | --- | --- |
| `loan_id` | INT | PK, Auto Increment | 借阅单唯一标识 |
| `copy_id` | INT | FK (book_copies.copy_id) | 借出书籍的指定副本 |
| `user_id` | INT | FK (users.user_id) | 借阅人标识 |
| `borrow_date` | DATE | NOT NULL | 借出日期 |
| `due_date` | DATE | NOT NULL | 应当归还的日期 |
| `return_date` | DATE | NULL | 实际归还日期 |
| `status` | ENUM | active/returned/overdue/lost | 当前借阅单状态 |
| `renewal_count` | INT | DEFAULT 0 | 续借总次数（隐性兴趣指标） |

### 2.2 预约记录表 (`reservations`)
用户预约已借出的书籍，确保在归还后能优先获得借阅权。
| 字段名 | 类型 | 约束 | 描述 |
| ------ | --- | --- | --- |
| `reservation_id` | INT | PK, Auto Increment | 预约唯一标识 |
| `book_id` | INT | FK (books.book_id) | 预约的图书 |
| `user_id` | INT | FK (users.user_id) | 预约人 |
| `reservation_date`| DATE | NOT NULL | 发起预约日期 |
| `expiry_date` | DATE | NOT NULL | 预约失效日期 |
| `status` | ENUM | pending/fulfilled/cancelled/expired | 当前预约状态 |

### 2.3 罚款记录表 (`fines`)
用于管理超期、损坏和丢失的费用。
| 字段名 | 类型 | 约束 | 描述 |
| ------ | --- | --- | --- |
| `fine_id` | INT | PK, Auto Increment | 罚款单标识 |
| `loan_id` | INT | FK (loans.loan_id) | 关联逾期/损坏的借阅单 |
| `user_id` | INT | FK (users.user_id) | 需要缴纳罚款的用户 |
| `amount` | DECIMAL(10,2)| NOT NULL | 罚款金额 |
| `reason` | VARCHAR(255)| NOT NULL | 罚款原因（如：逾期10天） |
| `date_issued` | DATE | NOT NULL | 开出罚单日期 |
| `date_paid` | DATE | NULL | 实缴日期 |
| `status` | ENUM | pending/paid/waived | 缴纳状态（未缴/已缴/豁免） |

## 3. 基础信息配置表

### 3.1 分类表 (`categories`)
无限极分类树，支持父子级图书排架。
| 字段名 | 类型 | 约束 | 描述 |
| ------ | --- | --- | --- |
| `category_id` | INT | PK, Auto Increment | 分类唯一标识 |
| `name` | VARCHAR(50) | UNIQUE, NOT NULL | 分类名称（如：计算机科学） |
| `parent_id` | INT | FK (categories.category_id) | 父级分类，NULL即顶级 |
| `description` | VARCHAR(255)| NULL | 类别说明描述 |

### 3.2 作者表 (`authors`) & 图书作者关联 (`book_authors`)
实现图书与作者的多对多关联，并支持作者排序（一作、二作）。
**Authors表**
* `author_id` (PK), `name`, `biography`, `birth_year`, `death_year`
**Book_Authors表**
* `book_id` (PK, FK), `author_id` (PK, FK), `author_order` (用于区分多作者顺序)

### 3.3 出版社表 (`publishers`)
* `publisher_id` (PK), `name` (UNIQUE), `address`, `contact_info`

## 4. 扩展与大数据智能表

### 4.1 用户行为记录表 (`user_behavior_logs`)
推荐算法的数据源，采集用户的浏览、搜索与交互数据。
| 字段名 | 类型 | 约束 | 描述 |
| ------ | --- | --- | --- |
| `log_id` | BIGINT| PK, Auto Increment | 日志追踪号 |
| `user_id` | INT | NULL | 操作用户（未登录时为空） |
| `book_id` | INT | NULL | 操作关联书籍 |
| `action_type` | ENUM | view_detail/add_to_shelf/click_preview/share | 交互行为枚举 |
| `duration_seconds`| INT | DEFAULT 0 | 页面停留时长 |
| `device_type` | VARCHAR(50) | NULL | 接入设备 |
| `ip_address` | VARCHAR(45) | NULL | 来源IP |

### 4.2 搜索历史记录表 (`search_history`)
分析整体用户的搜索热词（Trending Search）。
* `search_id` (PK), `user_id`, `keyword` (关键词), `result_count` (搜索命中返回数), `search_time`

### 4.3 书籍评论评分表 (`book_reviews`)
记录图书的UGC反馈，确保有借阅关联才能评价（真实性校验）。
* `review_id` (PK), `book_id` (FK), `user_id` (FK), `loan_id` (FK), `rating` (1-5), `comment_text`, `likes_count`, `status` (待审/通过/拒绝)

### 4.4 消息通知中心 (`notifications`)
系统触发的站内信或催还消息。
* `notification_id` (PK), `user_id` (FK), `type` (due_reminder/arrival_notice...), `title`, `content`, `is_read`, `send_time`, `read_time`
