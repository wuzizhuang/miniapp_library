# 总结与展望（论文第 9 章素材）

## 1 工作总结

本文围绕高校图书馆数字化管理的实际需求，设计并实现了一个基于 Spring Boot 和 Next.js 的智慧图书馆管理系统。系统采用前后端分离架构，以统一的 RESTful API 为核心，同时提供 Web、Android 和微信小程序三个前端接入渠道。经过完整的需求分析、系统设计、编码实现和测试验证，本系统在以下方面取得了较好的成果：

### 1.1 完整的业务闭环

系统实现了图书馆核心业务的完整状态流转：借阅 → 续借/归还/逾期/遗失 → 罚款自动生成 → 通知自动推送。预约排队系统支持副本归还后自动分配给队首用户并发送到馆通知。整个业务链路不依赖人工干预，实现了真正的自动化处理。

### 1.2 四端协同架构

系统后端提供 29 个 Controller、80+ 个 REST API 端点，三个前端（Web 43 页、Android 19 Screen、小程序 19 页）共享同一套 API，无需维护独立的接口版本。三种不同的前端技术栈（Next.js、React Native、原生微信小程序）在同一后端基础上运行，充分展示了多端开发的工程能力。

### 1.3 精细的权限控制

系统实现了完整的 RBAC 权限模型，从用户角色到细粒度权限（如 `loan:manage`、`fine:waive`），再到 `@PreAuthorize` 方法级控制，形成了"角色 → 权限 → API → 页面"的完整控制链。

### 1.4 丰富的增值服务

除核心借阅业务外，系统还实现了推荐动态（发布/点赞/关注）、座位预约（楼层/分区/时段/设施筛选）、服务预约（到馆还书/取书/咨询）、AI 智能助手等增值模块，提升了系统的实用性和创新性。

### 1.5 较高的测试覆盖

系统包含 51 个测试文件、356 个测试用例，通过率达 96.6%。测试覆盖了认证鉴权、借阅管理、预约处理、罚款管理、通知消息、角色权限控制等关键模块，为系统质量提供了有力保障。

## 2 主要创新点

1. **四端协同**：在一个毕业设计项目中同时交付四个平台（后端 + Web + Android + 小程序），且三端读者功能 29 项完全对齐，这在同类毕设中较为少见。

2. **业务自动化**：借阅逾期检测、罚款自动生成、预约自动分配、通知自动触发形成完整的业务自动化链条，不同于简单的 CRUD 系统。

3. **通知深链跳转**：系统支持 9 种通知类型的精准页面跳转，且在三个前端都已实现，提升了用户体验。

4. **搜索发现体验**：热门搜索 + 联想词 + 搜索历史的三位一体搜索发现体验，同样在三个前端都已落地。

## 3 不足与展望

尽管系统在功能完整性和工程质量上达到了预期目标，但仍存在以下不足和可改进之处：

### 3.1 已知局限

1. **邮件投递**：找回密码功能目前采用后端生成重置链接的模式，尚未接入真实的邮件发送服务（如 SMTP 或第三方邮件平台），实际部署时需补充此能力。

2. **前端自动化测试**：当前测试重点集中在后端接口层，前端尚未引入自动化测试框架（如 Cypress 或 Playwright），前端功能的回归验证依赖人工测试。

3. **性能优化**：系统尚未进行大规模性能测试和优化，如数据库查询优化、缓存策略调优、静态资源 CDN 分发等，在高并发场景下的表现有待验证。

4. **国际化支持**：系统目前仅支持中文界面，未实现多语言国际化方案。

### 3.2 未来展望

1. **微服务演进**：随着业务规模增长，可将单体后端拆分为借阅服务、通知服务、用户服务等微服务，提升系统的可扩展性和可维护性。

2. **全文检索**：引入 Elasticsearch 替代现有的数据库 LIKE 查询，提升图书检索的速度和相关性。

3. **智能推荐**：基于用户行为日志（已有 `user_behavior_logs` 表），利用协同过滤或内容推荐算法，为读者提供个性化的图书推荐。

4. **移动端管理功能**：为管理员提供移动端轻量管理入口，支持审批、通知处理等高频操作。

5. **容器化部署**：项目已有 `docker-compose.yml`，可进一步完善 CI/CD 流水线，实现自动化构建、测试和部署。

## 4 结语

本系统从需求分析到系统设计，从编码实现到测试验证，完成了一个具有实际使用价值的智慧图书馆管理系统。系统不仅覆盖了图书馆的核心业务流程，还通过多端接入、业务自动化、精细权限控制等特性，体现了软件工程的系统思维和工程实践能力。希望本项目的设计思路和实现经验能为同类系统的开发提供一定的参考价值。
