# 系统架构设计

## 1. 总体架构

## 2. 技术选型

| 层次 | 技术 | 选型理由 |
|---|---|---|
| 后端框架 | Spring Boot 3.x | 成熟生态、自动配置、企业级可靠性 |
| 后端语言 | Java 17 | LTS 版本，支持 record、sealed class 等特性 |
| ORM | Spring Data JPA + Hibernate | 声明式查询、自动 DDL、事务管理 |
| 认证鉴权 | Spring Security + JWT | 无状态认证、Token 续期、RBAC 权限模型 |
| 数据库 | MySQL 8.x | 事务支持、成熟稳定、社区活跃 |
| Web 前端 | Next.js 15 + React 18 + TypeScript | SSR 能力、Pages Router、类型安全 |
| UI 框架 | HeroUI + Tailwind CSS | 组件丰富、响应式设计、主题一致 |
| 数据请求 | SWR + Axios | 声明式数据获取、自动缓存与刷新 |
| Android | Expo + React Native + TypeScript | 跨平台、热重载、复用 JS 生态 |
| 微信小程序 | 原生 WXML + WXSS + JavaScript | 微信生态原生体验、API 直调 |

## 3. 后端分层架构

```
Controller（控制器层，29 个）
    │
    ▼
Service / ServiceImpl（业务逻辑层）
    │  ├─ 借阅规则校验
    │  ├─ 预约排队与分配
    │  ├─ 罚款自动生成
    │  ├─ 通知级联触发
    │  └─ 权限判断
    │
    ▼
Repository（数据访问层，Spring Data JPA）
    │
    ▼
Entity（实体层，映射数据库表）
```

## 4. 认证与权限模型

```
用户登录
    │
    ▼
JWT Token 签发（含 userId、roles、permissions）
    │
    ▼
前端存储 Token → 每次请求携带 Authorization Header
    │
    ▼
Spring Security Filter Chain
    ├─ JwtAuthenticationFilter：解析 Token、设置 SecurityContext
    ├─ @PreAuthorize：方法级权限控制
    └─ 角色体系：ADMIN / USER + 细粒度权限（loan:manage, fine:waive 等）
```

## 5. 多端接入策略

| 端 | 定位 | 覆盖场景 |
|---|---|---|
| Web（Next.js） | 主交付端 | 完整读者端 + 完整管理端 |
| Android（Expo/RN） | 移动读者端 | 核心读者链路 + 增值服务 |
| 微信小程序 | 轻量读者端 | 核心读者链路 + 增值服务 |

三端共享同一套后端 REST API，无需为不同客户端维护独立的接口版本。
