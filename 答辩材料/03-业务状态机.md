# 核心业务状态机

## 1. 借阅生命周期

```mermaid
stateDiagram-v2
    [*] --> ACTIVE : 读者发起借阅<br/>系统分配可用副本
    ACTIVE --> RETURNED : 读者自助归还<br/>或馆员确认还书
    ACTIVE --> OVERDUE : 定时任务检测<br/>超过 due_date
    ACTIVE --> ACTIVE : 续借成功<br/>renewal_count + 1<br/>due_date 延后
    OVERDUE --> RETURNED : 逾期归还<br/>自动生成逾期罚款
    ACTIVE --> LOST : 馆员标记遗失<br/>自动生成赔偿罚款
    OVERDUE --> LOST : 馆员标记遗失<br/>自动生成赔偿罚款
    RETURNED --> [*]
    LOST --> [*]
```

### 借阅业务规则
| 规则 | 说明 |
|---|---|
| 借阅上限 | 系统校验当前借阅数量 |
| 续借限制 | 最多续借 N 次，且该书无人预约排队时才可续借 |
| 逾期检测 | 定时任务每日检查，将超期的 ACTIVE 标记为 OVERDUE |
| 逾期罚款 | 逾期归还时自动计算罚款金额并生成 Fine 记录 |
| 遗失赔偿 | 标记遗失时按副本价格生成赔偿罚款 |
| 未缴罚款 | 有未缴罚款的读者不能借新书 |

## 2. 预约生命周期

```mermaid
stateDiagram-v2
    [*] --> PENDING : 读者发起预约<br/>没有可用副本时排队
    [*] --> AWAITING_PICKUP : 读者发起预约<br/>有可用副本直接分配
    PENDING --> AWAITING_PICKUP : 有副本归还<br/>系统自动分配给排队最前的读者
    AWAITING_PICKUP --> FULFILLED : 馆员确认取书<br/>生成借阅记录
    PENDING --> CANCELLED : 读者主动取消
    AWAITING_PICKUP --> CANCELLED : 管理员取消
    PENDING --> EXPIRED : 超过预约有效期
    AWAITING_PICKUP --> EXPIRED : 超过取书截止时间
    FULFILLED --> [*]
    CANCELLED --> [*]
    EXPIRED --> [*]
```

### 预约业务规则
| 规则 | 说明 |
|---|---|
| 副本分配 | 发起预约时若有可用副本，直接分配并进入 AWAITING_PICKUP |
| 排队顺序 | 多人预约同一本书时，按预约时间先后排队 |
| 到馆通知 | 副本分配成功后系统自动发送通知 |
| 取书期限 | AWAITING_PICKUP 状态有取书截止时间（pickup_deadline） |
| 履约校验 | 履约时校验读者未缴罚款和借阅上限 |
| 自动过期 | 定时任务检查超过有效期的预约 |

---

## 3. 罚款生命周期

```mermaid
stateDiagram-v2
    [*] --> PENDING : 系统自动生成<br/>逾期/遗失/损坏
    PENDING --> PAID : 读者缴纳<br/>或馆员柜台收款
    PENDING --> WAIVED : 管理员豁免
    PAID --> [*]
    WAIVED --> [*]
```

### 罚款触发场景
| 触发 | 类型 | 金额规则 |
|---|---|---|
| 逾期归还 | OVERDUE | 按逾期天数计算 |
| 标记遗失 | LOST | 按副本购入价格赔偿 |
| 损坏登记 | DAMAGE | 按评估金额 |

---

## 4. 馆藏副本状态

```mermaid
stateDiagram-v2
    [*] --> AVAILABLE : 新副本入库
    AVAILABLE --> BORROWED : 读者借出
    BORROWED --> AVAILABLE : 读者归还
    AVAILABLE --> RESERVED : 预约分配锁定
    RESERVED --> BORROWED : 预约履约取书
    RESERVED --> AVAILABLE : 预约取消/过期
    BORROWED --> LOST : 馆员标记遗失
    AVAILABLE --> DAMAGED : 馆员标记损坏
    LOST --> [*]
    DAMAGED --> [*]
```

## 5. 业务级联关系总图

```mermaid
flowchart TD
    A["读者借阅图书"] --> B["Loan: ACTIVE"]
    B --> C{"到期前归还?"}
    C -->|是| D["Loan: RETURNED"]
    C -->|否| E["Loan: OVERDUE"]
    E --> F{"归还?"}
    F -->|是| G["Loan: RETURNED<br/>+ 自动生成 Fine"]
    F -->|遗失| H["Loan: LOST<br/>+ 自动生成 Fine"]
    G --> I["Fine: PENDING"]
    H --> I
    I --> J{"处理?"}
    J -->|缴纳| K["Fine: PAID"]
    J -->|豁免| L["Fine: WAIVED"]
    D --> M["BookCopy: AVAILABLE"]
    M --> N{"有排队预约?"}
    N -->|是| O["Reservation: AWAITING_PICKUP<br/>+ 发送到馆通知"]
    N -->|否| M
    O --> P["馆员确认取书"]
    P --> Q["Reservation: FULFILLED<br/>+ 新 Loan: ACTIVE"]
```
