# 多端扩展设计与实现（论文第 7 章素材）

## 1 多端设计策略

### 1.1 设计理念

在完成 Web 端（读者端 + 管理端）的完整实现后，本系统进一步提供 Android 和微信小程序两个移动端读者入口。三端的设计策略如下：

| 端 | 定位 | 技术栈 | 覆盖范围 |
|---|---|---|---|
| Web | 主交付端 | Next.js + React + TypeScript | 完整读者端 + 完整管理端 |
| Android | 移动读者端 | Expo + React Native + TypeScript | 读者端核心链路 + 增值服务 |
| 微信小程序 | 轻量读者端 | 原生 WXML + WXSS + JavaScript | 读者端核心链路 + 增值服务 |

三端的核心设计原则是 **"共享统一后端 API，不为不同客户端维护独立接口版本"**。所有前端通过同一套 RESTful API 与后端通信，数据格式完全一致。

### 1.2 多端复用架构

```
           ┌─────────────┐
           │ Web (Next.js)│──── 读者端 + 管理端
           └──────┬───────┘
                  │
           ┌──────▼───────┐
           │  REST API    │ ← 统一后端（29 个 Controller, 80+ 端点）
           │ Spring Boot  │
           └──────┬───────┘
                  │
    ┌─────────────┼─────────────┐
    │             │             │
┌───▼───┐  ┌─────▼─────┐  ┌───▼───┐
│Android│  │   Web     │  │小程序 │
│Expo/RN│  │ Next.js   │  │原生   │
└───────┘  └───────────┘  └───────┘
```

管理端只在 Web 上实现。移动端聚焦读者场景，不复制管理功能。原因是图书馆管理操作涉及大量表格筛选、批量操作和复杂表单，适合在大屏幕设备上完成。

## 2 Android 客户端实现

### 2.1 技术选型

| 技术 | 说明 |
|---|---|
| Expo SDK | React Native 开发框架，简化构建和发布流程 |
| React Native | 跨平台 UI 框架，JavaScript 驱动原生组件渲染 |
| TypeScript | 类型安全开发 |
| React Navigation | 导航框架，处理 Tab 和 Stack 路由 |

## 5 多端技术方案对比

| 维度 | Web (Next.js) | Android (RN) | 小程序 (原生) |
|---|---|---|---|
| 渲染方式 | 浏览器 DOM | 原生 View 组件 | 微信 Webview + 原生 |
| 路由 | Pages Router | React Navigation | 微信原生页面栈 |
| 数据获取 | SWR + Axios | Axios | wx.request |
| 状态管理 | React Context + SWR | React Context | getApp() 全局状态 |
| 优势 | SSR 能力、大屏管理端 | 跨平台、热重载 | 微信生态、免安装 |

使用三种不同的前端技术栈实现同一套业务功能，充分展示了在统一后端 API 基础上进行多端开发的工程能力。
