## 4. 设计决策说明

> **为什么管理端只在 Web 实现？**
>
> 图书馆管理操作涉及大量表格筛选、批量操作和复杂表单，适合在大屏幕设备上完成。
> 移动端定位为读者入口，聚焦便捷的检索、借阅和通知体验。

> **为什么用三种不同的前端技术栈？**
>
> - Next.js (Web)：成熟的 SSR 框架，支持复杂的管理端和读者端双重场景
> - React Native (Android)：跨平台方案，复用 JavaScript 生态，一套代码可同时出 Android/iOS
> - 原生微信小程序：微信生态内无安装门槛，贴合"随时查书、快速预约"的轻量场景
>
> 三端展示了不同技术栈在同一后端 API 上的接入能力。
