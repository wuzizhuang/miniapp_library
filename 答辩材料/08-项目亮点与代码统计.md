## 2. 技术亮点

### 2.1 四端协同架构

- **统一后端** Spring Boot 3.x 提供 80+ 个 REST API
- **三个前端**（Web + Android + 微信小程序）共享同一 API
- **Web 独有**完整管理端（15 个 Dashboard 页面）
- 三种不同技术栈（Next.js / React Native / 原生小程序）展示多端开发能力

### 2.2 安全体系

| 能力 | 实现细节 |
|---|---|
| JWT 无状态认证 | Access Token + Refresh Token 双令牌 |
| 密码加密 | BCrypt 哈希 |
| 令牌黑名单 | 登出即时失效 |
| 请求限流 | 可选 Redis 支持的限流策略 |
| RBAC 权限模型 | 角色 → 权限 → @PreAuthorize 方法级控制 |
| CORS 安全 | 可配置多域名通配 |
| 异常屏蔽 | 500 错误不外泄堆栈，统一结构化响应 |

### 2.3 业务自动化

- **逾期检测**：定时任务自动标记过期借阅
- **罚款生成**：逾期归还 / 遗失自动生成罚款 + 通知
- **预约分配**：副本归还自动分配给队首预约用户 + 通知
- **预约过期**：定时任务自动过期超时预约

### 2.4 搜索发现体系

- **热门搜索**：基于搜索频率统计的热词推荐
- **联想词**：输入即时返回图书标题匹配建议
- **搜索历史**：登录用户的搜索记录保存与回放

### 2.5 通知深链跳转

支持 9 种通知类型（借阅提醒、预约到馆、罚款生成、评论审核等），
点击通知可直接跳转到对应业务详情页。三个前端均已实现。

### 2.6 增值创新模块

| 模块 | 说明 |
|---|---|
| 推荐动态 | 教师/管理员发布推荐，读者点赞、关注、评论 |
| 座位预约 | 按楼层/分区/时段查询，支持电源/靠窗筛选 |
| 服务预约 | 到馆还书/取书/馆员咨询，支持智能书柜方式 |
| AI 助手 | 可配置 OpenAI 接入，图书馆智能问答 |
| 行为日志 | 记录用户浏览、搜索、收藏行为，用于数据分析 |
